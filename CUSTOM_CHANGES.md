# LoRA Manager 增强版改动说明

本分支基于官方 [willmiao/ComfyUI-Lora-Manager](https://github.com/willmiao/ComfyUI-Lora-Manager) `v1.1.7`，准确基线提交为 `bf6a614e`。

改动重点是修复大型本地模型库中最影响使用体验的几个问题：Civitai 查询失败后无法重试、模型改名或移动后元数据失联、批量移动缺少反馈，以及缺少安全可靠的批量文件名整理能力。

## 主要增强

### 1. 可重试的 Civitai 元数据获取

- 新增“重试跳过项”按钮，只重新处理此前被标记为未找到的模型，不必重新扫描整个模型库。
- 强制重试时绕过旧的 `not found` 负缓存，避免失败一次后永久跳过。
- 修复元数据响应缺少 `sha256` 字段时直接报错的问题。
- 元数据 sidecar 缺失时，可优先复用缓存中的 SHA 和模型身份信息重新查询。
- 即使模型最初来自 Hugging Face，只要 SHA 能被 Civitai 识别，仍可补齐 Civitai 元数据。
- 保留已有模型身份字段，避免刷新 sidecar 时错误丢失版本信息。
- 核心元数据在下载预览图或视频之前先写入 sidecar，预览 CDN 故障不会丢掉本次刷新结果。
- 单个预览下载采用 15 秒总时限；超时后记录警告并继续，不再执行 60 秒 × 6 次的长时间重试。
- 预览属于可选资源，失败不会把已成功获取元数据的模型标记为刷新失败。

### 2. 安全智能改名

支持 LoRA、Checkpoint 和 Embedding 页面。

- 顶部按钮支持全库智能改名；选中模型后只处理选中项。
- 每张模型卡右上角提供单模型快速智能改名入口。
- 改名前显示预览，明确列出当前名称、建议名称和处理状态。
- 应用后可撤销本批次改名，撤销记录保存在本地历史中。
- 模型文件、预览图和 `.metadata.json` sidecar 会一起迁移。
- 已有工作流仍可能引用旧文件名，因此批量执行前应检查预览。

#### 文件名匹配规则

- 以当前 Civitai 页面展示的作者上传文件名为首选，不使用本地推断出的营销名或精度名。
- 必须通过 SHA256 精确匹配对应文件，避免同一模型不同精度、架构或量化版本串名。
- 同一 SHA 对应多个上传名时，按当前文件名的关键词覆盖度选择。
- 对 `I2V/T2V`、`FP8/BF16/FP16`、`Q4/Q6/Q8`、rank 等技术关键词执行冲突检查；存在明显矛盾时不自动改名。
- Civitai 名称为 `lora.safetensors` 等无描述性的通用名时保留当前名称。
- 目标文件已存在、多个模型会得到同一目标名或匹配不唯一时，不覆盖文件并标记为冲突或不变。

#### 性能与进度

- 全库和多选查询使用最多 3 路受控并发，兼顾速度与 Civitai 限流风险。
- 显示真实的 `已完成/总数 · 已用时间`，不再只有无法判断状态的转圈动画。
- WebSocket 进度包含独立任务 ID，多页面操作不会互相串进度。
- Civitai 文件页与 SHA 身份查询支持请求合并和内存缓存：成功结果缓存 15 分钟，失败结果仅缓存 60 秒，保留快速重试机会。
- 网络连接超时为 5 秒，单次请求最长 15 秒，避免少量失效项目拖住整批任务数分钟。
- 最终仍会对整批候选项统一执行路径占用和重名检查。

### 3. 改名与移动后的自动修复

- 扫描时根据唯一 SHA 识别在外部被改名或移动的模型。
- 自动将旧路径缓存记录关联到新路径，避免重新下载或重复查询元数据。
- 对同 SHA 的孤立 sidecar 执行安全合并，尽量保留标签、备注、收藏、触发词和 Civitai 身份。
- 若目标位置存在不同 SHA 的 sidecar，则拒绝合并，避免污染另一个模型的元数据。
- 缺失 SHA 时会重新计算，而不是直接跳过模型。

### 4. 批量移动进度反馈

- 内置“移动到文件夹”操作显示当前进度、总数和正在处理的文件。
- 多选移动按真实项目数更新进度。
- 已在目标文件夹的项目、部分失败和全部成功均有明确反馈。
- 移动结果会同步更新卡片路径和内部缓存。

## 新增接口

以下接口同时适用于 `loras`、`checkpoints` 和 `embeddings`：

- `POST /api/lm/{type}/smart-rename/preview`
- `POST /api/lm/{type}/smart-rename/apply`
- `POST /api/lm/{type}/smart-rename/undo`

智能改名预览请求可携带 `job_id`，进度通过现有 `/ws/fetch-progress` WebSocket 发送。

## 安全设计

- 智能改名默认只生成预览，不会直接修改文件。
- 不覆盖已存在的目标文件。
- 不删除发生冲突的模型。
- 改名前重新生成一次计划，避免预览后文件状态变化造成误操作。
- 撤销按相反顺序执行，并在原路径已被占用时停止恢复该项目。
- 只根据唯一 SHA 自动修复移动关系。

- 已在 WSL2 + ComfyUI + RTX 3090 的实际大型模型目录中验证。

## 与官方版兼容性

- 不修改 ComfyUI 核心文件。
- 保持原有 LoRA Manager 数据库、sidecar 和页面结构。
- 新功能建立在官方 v1.1.7 服务层和前端 API 结构之上。
- 后续合并官方更新时，应重点检查模型生命周期服务、批量元数据刷新流程、页面控制器和移动管理器的冲突。
